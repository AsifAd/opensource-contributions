## Problem

[PR #1553](https://github.com/jenkinsci/kubernetes-plugin/pull/1553) passes `--noReconnectAfter` to the inbound agent so the **jnlp** JVM exits after it cannot reconnect (e.g. build finished). That works when the pod has **only jnlp**.

Production-style agents often add sidecars:

- `docker` / `docker-daemon` (DinD)
- `sleep infinity` helpers
- custom tools containers

**Actual behaviour:** jnlp exits → sidecar still **Ready** → pod stays **Running** → cluster resource leak until manual cleanup.

**Related work logs:** multi-container pods (`jnlp`, `docker`, `docker-daemon`) on corp Jenkins — same shape as this bug class (agent timeout / pending containers), though #2809 is specifically post-build cleanup not ImagePullBackOff.

---

## Root cause (confirmed)

Kubernetes pod phase stays **Running** while any container is ready. `-noReconnectAfter` stops **jnlp** after the agent cannot reconnect; with a single container the pod completes and is removed. With **sidecars still Running**, the pod stays **Running** even after jnlp exits.

`Reaper.TerminateAgentOnContainerTerminated` calls `node.terminate()`, but `KubernetesSlave._terminate()` respects **pod retention** (`Always` → `shouldDeletePod` false). The pod is left behind with only sidecars running.

**Fix (branch `fix/no-reconnect-after-multi-container-2809`):**

- `PodUtils.isAgentContainerTerminated(pod, agentContainerName)`
- In `_terminate()`, if retention would keep the pod but the **agent container** has terminated → force `deletePod = true` (deletes whole pod, including sidecars). Covers #2809 and `Always` retention.

---

## Expected fix

When the **Jenkins agent container** terminates permanently:

1. Delete the **pod**, or  
2. Stop **all** containers in the pod template, or  
3. Add **liveness** semantics so a dead jnlp fails the pod

Must not break single-container pods (regression tests required).

### Pod template

- **jnlp:** `jenkins/inbound-agent` (recent tag)
- **sidecar:** `busybox` → `sleep infinity`
- **Label:** `k8s-multi-repro`

### Pipeline

```groovy
podTemplate(label: 'k8s-multi-repro') {
  node('k8s-multi-repro') {
    sh 'echo done'
  }
}
```

### Verify bug

```bash
export KUBECONFIG=$HOME/.kube/jenkins-oss-config
kubectl get pods -n jenkins -w
# After build: jnlp Exited, sidecar Running, pod Running
```
