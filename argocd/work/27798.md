## Problem

The ApplicationSet UI (list tile, table row, detail panel) exposed **no Refresh action**. Operators who push a new directory under a Git generator path had to force reconcile via kubectl:

```bash
kubectl annotate applicationset <name> \
  argocd.argoproj.io/application-set-refresh=true --overwrite
```

The controller **already** handles this annotation — Git generator bypasses revision cache when `RefreshRequired()` is true. The gap was API + UI only.

---

## Fix (PR #27932)

| Layer | Change |
|-------|--------|
| **Proto** | `Refresh` RPC — `POST /api/v1/applicationsets/{name}/refresh` |
| **Server** | `Server.Refresh` patches `argocd.argoproj.io/application-set-refresh: "true"`, RBAC `update` |
| **UI service** | `refreshApplicationSet(name, namespace)` |
| **UI** | Refresh on `appset-tile`, `appset-table-row`, detail action menu, `appset-resource-details` |
| **Tests** | `TestRefreshAppSet` (4 cases) + UI refresh helper tests |
