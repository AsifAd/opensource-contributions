## Problem

ApplicationSet UI had no Refresh action. Operators used:

```bash
kubectl annotate applicationset <name> \
  argocd.argoproj.io/application-set-refresh=true --overwrite
```

Controller already handles the annotation; gap was API + UI.

---

## Fix (PR #27932)

| Layer | Change |
|-------|--------|
| **Proto + server** | `Refresh` RPC → patch `argocd.argoproj.io/application-set-refresh` |
| **UI service** | `refreshApplicationSet(name, namespace)` |
| **UI surfaces** | `/applicationsets` list (tile + table), `/applications` mixed list (tile + table), detail action menu, resource details panel |
| **Follow-up** | Commit `f57aa5606` — wired Refresh through `ApplicationTiles` / `ApplicationsTable` (CI tsc fix) |
