## Problem

When an **ApplicationSet** lives outside the `argocd` namespace (e.g. `team-platform`) and the control plane manages that namespace via `application.namespaces`, the **ApplicationSet detail UI does not load**.

**Public repro (reporter):** https://cd.apps.argoproj.io/applicationsets/team-platform/services

---

## Root cause

`ApplicationsService.get()`, `resourceTree()`, and `watch()` sent query param **`appNamespace`** for ApplicationSet API calls. The server expects:

| Call | Correct param |
|------|----------------|
| GET, resource-tree | `appsetNamespace` |
| Watch stream | `appSetNamespace` |

See `server/applicationset/applicationset.proto` and `docs/operator-manual/applicationset/Appset-Any-Namespace.md`.

---

## Fix (PR #27931)

- `ui/src/app/shared/services/applications-service.namespace.ts` — maps namespace → correct query key
- `ui/src/app/shared/services/applications-service.ts` — uses helper in get / resourceTree / watch
- `ui/src/app/shared/services/applications-service.namespace.test.ts` — 4 unit tests
